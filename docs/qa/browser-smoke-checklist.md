# CainFlow 浏览器冒烟清单

每次修改 `启动链`、`执行链`、`节点绑定`、`媒体预览`、`设置面板` 或 `发布脚本` 后，都按下面步骤做一次浏览器验收。

## 建议顺序

1. 先运行 `pwsh ./scripts/run-local-regression.ps1`
2. 再启动源码版 CainFlow
3. 用 Codex 内置浏览器打开 `http://127.0.0.1:8767/`
4. 按下面清单逐项确认

## 脚本菜单 5 个功能分别是做什么的

如果你是从 `scripts/script-menu.bat` 里点选功能，可以按下面理解：

1. `本地回归测试`
   作用：一次性跑完整的本地基础门禁，等于“发布前校验 + 源码冒烟测试”组合。
   适合什么时候用：改了启动链、执行链、节点绑定、设置面板、媒体预览这类高风险代码后，优先先跑它。
   通过后说明什么：前端关键文件语法正常、后端 Python 能编译、回归工作流 JSON 没坏，而且源码版 CainFlow 至少能正常启动到首页。

2. `发布前校验`
   作用：做快速静态体检，不真正打开浏览器页面，重点查“代码本身有没有明显坏掉”。
   适合什么时候用：刚改完代码，想先快速确认语法和基础结构有没有问题时。
   通过后说明什么：`index.js`、执行链、媒体、节点、工作流等关键 JS 文件语法通过，`server.py` 和 `backend/` 编译通过，`workflows/regression/` 里的样例工作流 JSON 结构有效。

3. `源码冒烟测试`
   作用：直接启动当前仓库源码版 CainFlow，检查本地服务和首页初始化是不是正常。
   适合什么时候用：你想确认“代码能不能真的跑起来”，而不是只过语法检查时。
   通过后说明什么：本地 `http://127.0.0.1:8767/` 能正常返回页面，首页里最关键的 `toolbar`、`canvas-container`、`workflow-sidebar` 这些挂载点都存在，没有出现“服务启动了但页面白屏/没初始化”的明显问题。

4. `测试发布包（ZIP）`
   作用：解压已经打好的发布包，启动里面的 `CainFlow.exe`，检查打包产物能不能真正跑起来。
   适合什么时候用：本地打包完成后，或准备发给别人前，验证发布包不是“能生成但打不开”。
   通过后说明什么：发布 ZIP 内的程序、静态资源和启动链是完整的，至少能成功启动并把首页服务出来。

5. `打包 Windows 发布包`
   作用：调用本地打包脚本，生成可分发的 Windows 发布 ZIP。
   适合什么时候用：需要产出新的安装包/压缩包给自己验证，或准备发布新版本时。
   通过后说明什么：PyInstaller 构建、资源收集、发布目录整理和 ZIP 输出这条链路已走通，仓库当前状态可以产出一个新的 Windows 发布包。

## 固定检查项

1. 首页能在 10 秒内正常打开，没有白屏、报错弹窗或无限加载。
2. 工具栏、主画布、工作流侧栏都正常渲染。
3. 设置面板可以正常打开和关闭。
4. 历史面板或日志面板至少有一个可以正常打开和关闭。
5. 新增 1 个基础节点并拖动一次，确认画布没有卡死。
6. 刷新页面后，应用仍能重新初始化。

## 建议补充检查

1. 导入 `workflows/regression/` 下的样例工作流，确认能正常加载。
2. 如果本次修改涉及执行链，至少打开 1 个文本链和 1 个图片/视频模板做结构检查。
3. 如果本次修改涉及设置面板，顺手检查代理设置、模型设置和通用设置页签切换。

## 发布前补充

1. 本地构建发布包后，先运行 `pwsh ./scripts/smoke-test-cainflow.ps1 -Mode release -ZipPath <发布包路径>`
2. 再用 Codex 内置浏览器打开本地 CainFlow，复核上面的固定检查项
