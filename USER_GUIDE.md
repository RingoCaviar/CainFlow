# CainFlow 使用手册

这份手册只讲一件事：第一次打开 CainFlow 之后，怎样把它跑起来，并成功生成第一张图片。

## 第一步：启动 CainFlow

如果你使用的是发布版：

1. 解压下载好的压缩包。
2. 双击 `CainFlow.exe`。
3. 等待浏览器自动打开 `http://127.0.0.1:8767`。

如果浏览器没有自动打开，就手动访问 `http://127.0.0.1:8767`。

## 第二步：打开设置面板

进入主界面后，先打开设置面板。

你需要在这里完成两件事：

1. 添加 API 供应商。
2. 添加模型，并把模型绑定到供应商。

只有这两步配置完成后，图片生成节点里才会出现可选模型。

## 第三步：添加 API 供应商

在设置面板里找到供应商配置区域，然后新增一个供应商。

需要填写的内容通常有这几项：

- **供应商名称**：随便起一个你看得懂的名字，比如 `OpenAI`、`Gemini`、`我的中转站`。
- **API 密钥**：填写供应商给你的 Key。
- **API 地址**：填写供应商的接口基址，不要填官网首页。
- **自动补全**：建议保持开启。CainFlow 会根据模型类型自动补全请求路径。

### 常见填写示例

如果你用的是 OpenAI 兼容接口：

- API 地址一般类似 `https://api.openai.com`
- 也可能是第三方中转地址，例如你自己供应商提供的域名

如果你用的是 Gemini / Google 接口：

- API 地址一般类似 `https://generativelanguage.googleapis.com`

### 这里最容易填错的地方

- 不要把 API 地址填成官网首页。
- 不要漏掉 `http://` 或 `https://`。
- 如果是第三方中转平台，要确认这个平台真的支持你后面要用的图片模型。

填完以后保存。

## 第四步：添加模型

供应商配置好之后，接着去模型配置区域添加模型。

你有两种方式：

### 方式一：自动获取模型列表

如果供应商支持读取模型列表：

1. 在该供应商卡片上点击“获取模型列表”。
2. 等待 CainFlow 拉取模型。
3. 在弹出的列表里选择你要用的模型并添加。

这是最省事的方式，适合大多数情况。

### 方式二：手动添加模型

如果供应商不支持自动拉取，或者你已经知道模型名，就手动添加。

你需要填写：

- **模型名称**：显示在界面里的名字，可以自己起，例如 `生图-gpt-image-2`。
- **模型代码（Model ID）**：供应商文档里的真实模型 ID，例如 `gpt-image-2`。
- **绑定供应商**：选择你刚才添加的那个供应商。
- **模型用途**：如果这个模型是用来生成图片的，这里一定要选 `生图`。

这一项很关键。

如果模型用途不是 `生图`，它就不会正确出现在图片生成节点里。

## 第五步：检查模型是否配置成功

配置完成后，可以简单确认一下：

1. 回到画布。
2. 新建一个“图片生成”节点。
3. 查看节点里的模型下拉框。

如果你刚刚添加的模型已经出现在下拉框里，说明供应商和模型配置成功了。

如果这里还是空的，通常是下面这几种原因：

- 模型没有绑定供应商。
- 模型用途没有选 `生图`。
- 供应商没有保存成功。
- API 地址或配置有问题，导致模型列表虽然加了，但实际不可用。

## 第六步：创建第一条生图流程

现在开始正式生图。

建议第一次只用 3 个节点：

1. 文本输入
2. 图片生成
3. 图片预览

### 具体操作

1. 在画布空白处右键。
2. 添加一个“文本输入”节点。
3. 再添加一个“图片生成”节点。
4. 再添加一个“图片预览”节点。

然后把它们连起来：

- 文本输入 -> 图片生成
- 图片生成 -> 图片预览

这就是最基础、最稳定的第一条生图链路。

## 第七步：填写生图参数

在“文本输入”节点里输入你的提示词。

例如：

```text
a cinematic portrait of a girl, soft lighting, detailed hair, high quality
```

然后打开“图片生成”节点，重点检查下面几项：

- **模型**：选择你刚才配置好的图片模型。
- **分辨率**：先用常规尺寸测试，例如 `1024x1024`。
- **质量**：如果你使用的是 OpenAI / GPT 图片模型，可以选 `auto`、`low`、`medium` 或 `high`。

第一次建议这样选：

- 分辨率用 `1024x1024`
- 质量用 `auto`

这样更稳，也更容易先测通流程。

## 第八步：开始生成图片

确认连线和参数都没问题后，点击运行。

正常情况下：

1. 图片生成节点开始请求。
2. 节点顶部会显示运行状态点。
3. 生成完成后，图片会出现在图片预览节点中。

点击预览图可以进入大图查看。

## 第九步：如果生成失败，先看这几个地方

如果第一张图没有成功出来，优先检查下面几项：

### 1. API Key 是否正确

- 有没有填错
- 有没有多余空格
- 这把 key 是否真的属于当前供应商

### 2. API 地址是否正确

- 是否填写成了官网首页
- 是否漏掉了协议头
- 是否和当前供应商文档一致

### 3. 模型和供应商是否匹配

例如：

- `gpt-image-2` 应该挂在 OpenAI 兼容供应商上
- Gemini 模型应该挂在 Gemini / Google 对应接口上

### 4. 模型用途是否选成了“生图”

如果模型用途填错，图片生成节点通常就不会正常工作。

### 5. 当前供应商是否真的支持这个图片模型

很多中转平台并不是所有图片模型都支持，尤其是图片生成和图片编辑接口，支持情况可能不一样。

## 第十步：跑通后你可以继续做什么

当第一张图已经能正常出来后，你就可以继续扩展流程了，比如：

- 在图片生成后面接“图片保存”节点
- 一次生成多张图
- 打开自动重试
- 用历史记录面板回看结果
- 给图片生成节点增加参考图

建议先把最基础的“文本输入 -> 图片生成 -> 图片预览”跑通，再往后加功能，这样最不容易排错。
